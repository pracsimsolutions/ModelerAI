# ModelerAI knowledge base — INDEX

Two-layer KB. The DLL parses this file at module init.

1. **Auto-generated reference** — every FlexScript class + every FlexScript command, extracted from FlexSim's shipped XML help files via `tools/build-kb.js`. NOT listed in the manual table below — discovered through `INDEX.json` (BM25 stats) and fetched from `topics/classes/<Class>.md` or `topics/commands/<command>.md` on demand.
2. **Hand-written topics** — pattern guides, system-level concepts, hard-won knowledge that isn't in any XML. Listed in the table at the bottom of this file.

## Layout

```
KNOWLEDGE/
  INDEX.md                          (this file — hand-written topics + the directive)
  INDEX.json                        (auto: BM25 stats + topic metadata, never sent to LLM)
  catalog/
    classes.md                      (auto: always-on table — 116 classes grouped by module)
    commands-active.md              (auto: always-on table — 790 active commands, no deprecated)
    commands.md                     (auto: full reference incl. ~426 deprecated; on-demand only)
  topics/
    classes/<Class>.md              (auto: one per class, fetched on demand via search_kb / read_topic)
    commands/<command>.md           (auto: one per command)
    patterns/                       (hand-written: pattern guides — AGV system build, port wiring, …)
    concepts/                       (hand-written: cross-cutting concepts)
    *.md                            (legacy hand-written; migrating to patterns/)
```

Regenerate the auto layer with: `node tools/build-kb.js`. Inputs:
- `<FlexSim>/help/FlexScriptAPIReference/**/*.xml`
- `<FlexSim>/help/CommandReference/Commands.xml`

`build-kb.js` flags deprecation via group code 31 ("Deprecated") and extracts the replacement command/method from the description body.

## Always-on cached prefix

`catalog/classes.md` (~3K tokens) and `catalog/commands-active.md` (~18K tokens) are stable across turns and intended to live inside the Anthropic `cache_control: ephemeral` system prefix. Do NOT include the auto-generated content in any volatile portion of the prompt — that invalidates the cache.

## Known coverage gaps (post-reset 2026-06-02)

The hand-curated layer was nuked on 2026-06-02 to start fresh after pollution from rapid iteration. Everything below is now "open" — rebuild modeler-led, one topic per learning session, per `docs/learning-sessions.md`.

- **Per-class property schemas** under `KNOWLEDGE/schemas/` (Source, Queue, Processor, Sink, Operator, ForkLift, TaskExecuter, Conveyor, Combiner, Separator, MultiProcessor, Rack, BasicFR, NetworkNode, ControlPoint, Path). One JSON file per class. Drives `modelerai_set_property`.
- **Per-class config topics** under `KNOWLEDGE/topics/` — prose context that pairs with each schema (when to use what, common patterns, gotchas).
- **Picklist-family routing** — when the AI needs a trigger/parameter-onset/PM body, which picklist family applies. Add only as scenarios are validated against real prompts.
- **Manual dedup pass.** `tools/dedup-manual.js` is scaffolded; consolidates redundant restatements across the 124 manual topics. ~$2-5 in API costs. Defer until needed.

What's still trusted:
- Auto-generated `topics/classes/<Class>.md` (116) and `topics/commands/<command>.md` (1216) — derived from FlexSim's XML help, regenerated by `tools/build-kb.js`.
- Auto-generated `topics/manual/<id>.md` (124) — scraped from FlexSim's manual.
- `catalog/classes.md`, `catalog/commands-active.md`, `catalog/index.md` — always-on cached prefix.
